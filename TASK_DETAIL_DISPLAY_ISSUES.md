## 修复方案

### 后端数据检查
1. 确认数据库中是否正确保存了所有字段
2. 确认API返回的数据是否完整

### 前端显示修复
1. **商家中心任务详情页** (`frontend/src/app/merchant/tasks/[id]/page.tsx`)
2. **管理后台任务详情弹窗** (`frontend/src/app/admin/tasks/page.tsx`)
3. **买手任务详情页** (`frontend/src/app/tasks/[id]/page.tsx`)

### 需要修复的字段映射
- `goodsList[0].orderSpecs` → 下单规格显示
- `goodsList[0].num` → 下单数量
- `count` → 任务单数（当前显示错误）
- `keywords` → 关键词列表（主关键词、货比关键词、备用关键词）
- `hasSubProduct` → 控制副商品浏览时长显示
- `memo` → 下单提示
- `contactCSContent` → 联系客服内容
- `praiseList` → 好评文字内容
- `isPasswordEnabled` + `checkPassword` → 口令验证
- `extraCommission` / `addReward` → 额外赏金
- `fastRefund` → 快速返款
- `unionInterval` / `orderInterval` → 接单间隔
- `weight` → 包裹重量

## 优先级
1. **P0 - 严重错误**：商品本金显示¥0.00、任务单数显示错误
2. **P1 - 重要缺失**：关键词详情、下单规格、好评内容
3. **P2 - 一般缺失**：增值服务详情、下单提示
